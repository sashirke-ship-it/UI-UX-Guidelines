## Quick map — what lives where

| **Section** | **Use it for** | **Don't put here** |
| --- | --- | --- |
| 📐 Standards | Topic-based design rules (scope, brand, tokens, components, style card, governance) | Per-project work, meeting notes, rollout plans |
| ⚙️ Process & Governance | How standards ship, compliance enforcement, deviation log, exception process | Topic rules themselves (those live in Standards) |
| 🔁 Workflow & Tooling | Claude Design → Claude Code → Figma → GitHub loop, design catalog, Github workflows, consolidation map | Project-specific deliverables |
| 📦 Projects | Per-project UI/UX compliance work, project-level audits | Topic-level standards or process docs |
| 🗂️ Planning & Meetings | Time-bound rollout plans, meeting notes, legacy proposals pending consolidation | Evergreen rules or workflow docs |

## How to use this hub

- **📐 Standards** — topic-based design rules. Canonical numbered documents (scope, brand, components, tokens, etc.).
- **⚙️ Process & Governance** — how standards ship, how compliance is enforced, and how deviations are logged.
- **🔁 Workflow & Tooling** — the Claude Design → Claude Code → Figma → GitHub loop and supporting reference indices.
- **📦 Projects** — per-project UI/UX compliance work. Projects only; no topic-level standards live here.
- **🗂️ Planning & Meetings** — time-bound rollout plans, legacy proposals pending consolidation, and meeting notes.

## Organizing principle

<aside>
🧭

**Topics** (Standards, Process & Governance, Workflow & Tooling) and **Projects** live in separate branches — never mixed. Time-bound planning and meeting docs live in their own branch so they don't clutter the evergreen spine.

</aside>
