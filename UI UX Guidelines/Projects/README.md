# Projects (pilots, prototypes, compliance notes)

Everything in this directory is **historical, experimental, or product-specific work**. It is **not** the binding company design system unless a file explicitly says it was promoted into **`Standards/`** or the **hub**.

## How to use this folder

- **Engineers / designers:** implement from the **hub** + **`UI UX Guidelines/Standards/`** + root **`DESIGN.md`** / **`AGENTS.md`**, not from a subfolder here by default.
- **Stitch / AI tools:** use **repository root** [`DESIGN.md`](../../DESIGN.md) for org-wide tokens. Any `DESIGN.md` **inside** a project subfolder is **local to that pilot** (see banner on those files).
- **Compliance / planning:** MD files here may reference delivery status, screenshots, or HTML exports for a single initiative—treat them as artifacts, not canonical tokens.

## When you add material

Add a short **banner** at the top clarifying pilot vs promoted standard, and link back to [`../UI UX guidelines 20be6c5f913d4f53ac0e76eb5e904eef.md`](../UI%20UX%20guidelines%2020be6c5f913d4f53ac0e76eb5e904eef.md) when you want readers to reconcile with the teal semantic palette (`#208692` primary, etc.).
