## Goal

Validate that teams can ship a real feature **using only** this standards hub + Figma + code.

## Pick a pilot

- **Scope:** one squad, one milestone-sized feature (2–4 weeks).
- **Success:** fewer ad-hoc components; faster design QA; no a11y regressions found late.

## Week 0 — setup

- Assign a **pilot lead** (designer or PM).
- Confirm **Figma library** and **Storybook** links in **Inputs & prerequisites**.

## Week 1 — design

- Start components from library; log **gaps** as proposals in Governance.
- Run **heuristic pass** with Experience principles.

## Week 2 — build

- Implement with tokens; keyboard test early.

## Week 3 — review

- Use **Release review checklist**.
- Capture **anti-patterns** avoided and any **exceptions** (with owner approval).

## Retro questions

- What was missing from the docs?
- Which page should merge or split?
- Update **Standards changelog** with pilot outcomes and v1.1 tasks.
