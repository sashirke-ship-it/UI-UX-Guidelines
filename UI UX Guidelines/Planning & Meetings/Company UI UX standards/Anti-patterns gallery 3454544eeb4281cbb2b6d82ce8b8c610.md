# Anti-patterns gallery

## Purpose

Capture **real examples** from internal products (with permission) showing the wrong pattern and the **approved fix**.

## How to add an entry

- Screenshot **before / after** (blur PII).
- Link to principle violated and the **correct pattern** page.
- Severity: **blocker / should-fix / polish**.

## Starter examples (replace with your own)

1. **Icon-only buttons with no tooltip** → add text label or accessible name.
2. **Red and green only for status** → add icon + text.
3. **11px body text** → use minimum body token from **Brand & foundations**.
4. **Endless modal stacks** → flatten flow; use page or drawer pattern.
