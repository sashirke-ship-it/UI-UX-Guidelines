# Surface addenda (multi-channel)

## Why addenda exist

Web patterns may **not** transfer to email, kiosk, print, or slides. Capture deltas here **per surface**.

## Template — copy per surface

### Surface name (e.g. Transactional email)

- **Viewport / constraints:** max width, image blocking, font support
- **Interaction model:** links only vs rich interactions
- **Typography limits:** webfonts available? fallback stack?
- **Color / contrast:** different background assumptions
- **Components allowed:** which web components **do not** apply
- **Owners:** who approves ship for this surface

### Kiosk / large touch

- Target distance, timeout, attract loop policy

### Print / PDF

- Margins, pagination, link handling

### Internal tools

- May relax visual polish but **not** accessibility for employee disabilities
