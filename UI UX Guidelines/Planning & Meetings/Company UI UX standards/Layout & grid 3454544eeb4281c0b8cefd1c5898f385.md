# Layout & grid

## Page structure

- **App shell:** top nav vs sidebar; max content width for reading vs tools.
- **Sticky regions:** primary actions or filters may stick; avoid double sticky bars.

## Grid and columns

- **12-column grid** (common) or **10-column** for data-heavy apps — pick one.
- **Gutters:** scale up at larger breakpoints.

## Breakpoints (example — calibrate to analytics)

- **xs** < 480 — phones portrait
- **sm** 480–767 — large phones / small tablets
- **md** 768–1023 — tablets / small laptops
- **lg** 1024–1439 — laptops
- **xl** ≥ 1440 — large monitors

## Responsive rules

- **Content first:** reflow before hiding; if hiding, provide another path.
- **Touch targets:** minimum **44×44 px** visual target on touch surfaces.
- **Safe areas:** respect iOS/Android safe area insets for notched devices.

## Alignment

- Left-align LTR reading text; numeric columns **right-align**.
- Form labels: **top labels** for complex forms; inline only when space-proven.

## Density

- **Comfortable** default; **compact** only for power-user tables with toggle and a11y review.

## Print / PDF / kiosk addenda

For non-web surfaces, see **Surface addenda** for viewport and interaction constraints.
