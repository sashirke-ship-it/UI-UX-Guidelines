# Onboarding — start here

## First day with the standards

1. Read **Introduction & scope** (10 min).
2. Skim **Experience principles** (10 min).
3. Open **Design tokens** + **Components & patterns** relevant to your squad (20 min).
4. Bookmark **Accessibility** and **Content & UX writing** for reviews.
5. Complete **Inputs & prerequisites** checklist with your PM or design lead.

## For engineers

- Map tokens to your theme file; install or link Storybook stories for primitives.

## For designers

- Pin the Figma library; use semantic styles only; file gaps in Governance.

## For PMs

- Use **Release review checklist** before major launches.
