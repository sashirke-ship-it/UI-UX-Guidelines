# Governance

## Roles

- **Document owner:** breaks ties; schedules reviews.
- **Stewards:** design, engineering, accessibility, content — see **Introduction & scope**.

## Contribution workflow

1. **Proposal:** problem statement, user impact, proposed pattern/token change, rollout plan.
2. **Review:** steward(s) + owner; design QA on Figma; eng spike if needed.
3. **Ship:** update Figma library, code tokens/components, and **Notion** pages.
4. **Communicate:** announce in team channel with **what changed** and **deadline for adoption**.

## Deprecation

- Mark old pattern **deprecated** for one release; link replacement; remove after deadline.

## Cadence

- **Monthly** standards review (30 min) + **changelog** entry.

## Figma and code linkage

- Component names mirror code; token names shared across tools.
