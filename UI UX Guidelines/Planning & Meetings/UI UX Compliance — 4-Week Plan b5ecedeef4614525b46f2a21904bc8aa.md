## PM Operating System — Where UI/UX Reporting Fits

The PM Operating System runs on two mandatory weekly cadences that transfer with the Steward PM rotation:

### Monday PM Meeting

- **When:** Every Monday at 10:00 AM EST
- **Facilitator:** Steward PM for the week
- **UI/UX agenda items:** Surface blockers, report wireframe progress, flag projects with zero wireframes or missing WCAG declarations

### Friday EOD Readiness Check

- **When:** Every Friday before end of day
- **Owner:** Steward PM for the week
- **UI/UX compliance checklist:**
    - [x]  UI/UX status section complete in RAD log
    - [x]  All open UI/UX decisions logged with owners and due dates
    - [x]  Blockers flagged with assigned owners

### RAD Log — UI/UX Status Section (Mandatory Fields)

| **Field** | **Description** |
| --- | --- |
| Prototype Status | Not Started / In Progress / Complete |
| Screens Completed | Count of wireframed/prototyped screens |
| Screens Remaining | Count of screens still needing wireframes |
| Interaction States Coverage | % of primary screens with loading/empty/error/success states designed |
| Accessibility Compliance | WCAG target declared (Yes/No) + current Lighthouse score if available |
| Open UI/UX Decisions | Count of open decisions with owners and due dates |
| Stakeholder Sign-off | Pending / Approved |
