## Component system (ShadCN)

- Use ShadCN components by default.
- Customize through **theme tokens** and variants; avoid one-off CSS.
- Prefer **composition** over forking.

### Button types

- **Primary:** solid primary, white text — main actions.
- **Secondary:** outline (complementary), text deep primary — alternative actions.
- **Ghost/Tertiary:** transparent — subdued actions.
- **Destructive:** solid red (ShadCN default) — see **Brand & foundations** semantic rules.

### Button states and spacing (document in Figma)

Every variant: **default, hover, focus** (visible ring, token-colored), **active/pressed**, **disabled**, **loading** (spinner + reduced interaction), **selected/toggle** when applicable.

- Hover: token-based fill/stroke (no ad-hoc hex).
- Spacing: document **padding** for Dev Mode; prefer **8pt-grid** values (e.g. **32px** horizontal for large CTA unless exception filed).
- **Touch targets** ≥ 44×44px (may need invisible hit area beyond visual button).

### Button anatomy (8pt grid)

- Heights: **Large 48px**, **Medium 40px**, **Small 32px**
- Minimum touch target: **44×44px**

### Primary state matrix (token-aligned)

- Default **#208692** · Hover **#164F5B** · Pressed **#145A66** · Loading **#208692** (spinner)

### Destructive matrix

- Enabled **#DC2626** · Hover **#B91C1C** · Pressed **#991B1B** · Disabled **#D1D5DB** (text **#9CA3AF**) · Soft **#FEE2E2**, **#FECACA**

### Controls (forms + chips)

- Checkbox/radio selected: **#208692** · Switch active: **#208692**
- Chips: Primary **#208692**, Success **#16A34A**, Error **#DC2626**, Warning **#F59E0B**, Disabled **#D1D5DB**

### Inputs and selects

- Focus ring/border: **primary #208692** · Error: **destructive #DC2626**
- Dropdown: item height **36px**, padding **12px**, shadow **lv2**

### Platform-specific and forks

- **One org-wide SSOT**; platform-specific guidance only when shared across products.
- **Fork required:** document why, what changed, where used, maintenance risk — link in PR (see **Governance**).

## Required states (non-optional)

Every screen/component defines:

- **Loading** (skeleton/spinner, avoid layout shift)
- **Empty** (why empty + what to do next)
- **Error** (recovery + retry)
- **Success** (toast/banner; what happens next)

## Approved pattern examples (from § 17)

- **Dashboard:** KPI cards + table + detail drawer
- **Table + filters + drawer:** filter bar + table + empty/error/loading
- **Form:** inline validation + disabled submit until valid + success toast
